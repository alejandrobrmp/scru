description: Wrap completed work and open the draft PR
---

Wrap the completed task work using the current branch changes, the task spec, and the latest implementation details.

Use the `wrap-task` skill.

Requirements:

- Inspect the changes made since the last task update.
- Summarize the implementation clearly.
- Update the task spec if the recorded task details need to change.
- Save any important technical detail to memory.
- Commit the final changes to the current branch.
- Open or refresh the draft pull request.

If the work is not actually complete, stop and ask what remains before wrapping.
